NCF Management

Complete NCF (Número de Comprobante Fiscal) Management for Dominican Republic

Overview

Fiscal Accounting (Rep. Dominicana) implements the administration and management of fiscal receipt numbers (NCF) for compliance with norm 06-18 of the Dominican Republic Internal Revenue Service (DGII). This module ensures full compliance with DGII regulations for fiscal documentation and electronic invoicing.

Key Features

Automatic NCF Generation

The module automatically generates NCF numbers for all invoices based on customer type and transaction nature. It supports all 17 NCF types required by Dominican Republic law, including fiscal credit invoices (01), consumer invoices (02), credit and debit notes (03-04), and special regimes (11-17).

Pre-configured sequences ensure proper numbering and compliance with DGII requirements. The system validates NCF format in real-time and prevents duplicate or invalid numbers.

DGII Web Service Integration

Real-time validation with DGII web services ensures that all NCF numbers are properly validated against the official database. The module checks NCF-RNC relationships and verifies the status of fiscal documents automatically.

Automatic synchronization keeps your fiscal data up-to-date with DGII requirements, reducing the risk of compliance issues and audit problems.

Sequence Management

Configure and monitor NCF sequences directly from the journal settings. The module provides comprehensive sequence management including authorization tracking, expiration monitoring, and automatic sequence assignment based on customer classification.

Visual indicators show sequence usage and remaining numbers, helping you plan for sequence renewals and avoid interruptions in fiscal operations.

How to Create NCF Sequences (Fiscal Sequences)

Step-by-step guide to create Fiscal Sequences (account.fiscal.sequence):

Step 1: Access Fiscal Sequences

Navigate to Accounting > Configuration > Localization > Fiscal Sequences

Step 2: Create New Sequence

Click the "Create" button to add a new fiscal sequence

Step 3: Configure Basic Information

  • Name: Descriptive name for the sequence (e.g., "NCF 01 - Credit Invoice 2024")
  • Fiscal Type: Select the appropriate NCF type (01-17)
  • Company: Select your company
  • State: Leave as "Draft" until ready to activate

Step 4: Set Sequence Range

  • From Number: Starting number of your NCF range (e.g., 00000001)
  • To Number: Ending number of your NCF range (e.g., 01000000)
  • Next Number: Will be set automatically to the "From Number"

Step 5: Set Authorization Details

  • Authorization Date: Date when DGII authorized this sequence
  • Expiration Date: When the sequence expires (usually 2 years from authorization)
  • Authorization Number: Official authorization number from DGII

Step 6: Configure Usage

  • Warning Percentage: Set a percentage (e.g., 90%) to receive warnings when sequence is nearly exhausted
  • Fiscal Journals: Assign this sequence to specific accounting journals

Step 7: Activate Sequence

Once all information is complete, change the State to "Active" to start using the sequence

Important Notes:

  • Each NCF type (01-17) requires its own sequence
  • Sequences must be authorized by DGII before activation
  • Monitor sequence usage regularly to avoid running out of numbers
  • Request new sequences from DGII before current ones expire
  • The system will automatically warn you when sequences are nearly exhausted

Customer Classification & RNC Management

Automatic customer classification determines the appropriate NCF type based on customer information. The module validates RNC (business tax ID) and Cedula (personal ID) formats and integrates with DGII services to verify customer data.

Support for final consumers, business customers, government entities, and export transactions ensures proper fiscal treatment for all types of operations.

Exchange Rate Management

Automatic import of official exchange rates from the Dominican Republic Central Bank ensures accurate currency conversion for multi-currency transactions. The module maintains historical exchange rates for proper fiscal reporting and compliance.

Daily updates and manual rate adjustments provide flexibility while maintaining compliance with official exchange rate requirements for fiscal purposes.

Configuration

Configure your company's fiscal information and NCF sequences through Accounting > Configuration > Journals. Set up customer classifications and tax configurations to ensure proper fiscal treatment of all transactions.

The module includes comprehensive documentation and setup wizards to guide you through the initial configuration process.
